# Full Sweep Scenario

Run this scenario with GLM Code from the testbed root:

1. Read `README.md`, `docs/product.md`, and `src/taskStore.mjs`.
2. Run `npm run check`.
3. Use search to find every reference to `status`.
4. Call the MCP tool `testbed_task_count`.
5. Add a `priority` field to tasks with default `normal`.
6. Update Node tests, sample data, and docs.
7. Run `npm run check` again.
8. Report changed files and whether ignored files were left untouched.
